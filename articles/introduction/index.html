本站是建立在Github Page上的静态网站，前端使用Materialize实现，文章数据使用KuroArticle生成。

KuroArticle使用简介

首先在KuroArticle的目录下建立配制文件config.json。此文件包含写作目录和输出目录两个字段。如下：

{
    "articlesDirectory" : "D:\\MyBlog\\articles",
    "outputDirectory" : "D:\\MyBlog\\devel\\articles"
}

跟make的行为类似，KuroArticle会自动扫描写作目录下的文章变化。

写作目录的结构如下，每篇文章的目录下下都有简介brief.json与文章正文article.md两个文件，文章封面图、正文中引用到的图片都应放在文章的目录中。

写作目录
  ├─文章目录1
  │   ├─brief.json
  │   ├─article.md
  │   ├─文章引用图片a.jpg
  │   └─文章引用图片b.png
  │
  ├─文章目录2
  │   ├─brief.json
  │   ├─article.md
  │   ├─文章引用图片c.jpg
  │   └─文章引用图片d.png
  │
  └─文章目录3 ...

article.md与普通的Markdown文件没有区别。brief.json的格式为：

{
    "coverImage"        : "cover-image.jpg",
    "title"             : "Introduction of Uiharu.Hime",
    "subtitle"          : "网站与KuroArticle的使用说明",
    "summaryTitle"      : "这是一个简单的介绍",
    "summaryContent"    : "同时还是一个使用的备忘录",
    "creationTime"      : "2016-12-8 12:10",
    "tag"               : "no-tag"
}
